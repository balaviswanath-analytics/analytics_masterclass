The Reference Customer Effect: Causal Network Analysis

When one customer churns, their contacts churn at 2x the baseline rate. Is this because churn is contagious, or because connected customers share the same risk factors? This week introduces instrumental variables for peer effects and serves as the Q1 capstone — synthesizing all 13 weeks into one integrated system.

Part 1: When Customers Influence Each Other

A B2B SaaS platform notices something alarming: when a "reference customer" (featured in case studies) churns, other customers in that reference customer's network begin churning at elevated rates. The reference customer might only contribute $200K in annual revenue. But their indirect value — influence on prospects, peer validation, referral pipeline — could be worth $8M in influenced revenue. Standard CLV models completely miss this network CLV.

Three Types of Network Effects in Churn

EffectMechanismExample
ContagionOne customer's behavior directly influences another'sColleague tells you they switched; you consider switching too
HomophilyConnected customers share characteristics driving similar behaviorCompanies in the same industry face the same budget pressures
Common exposureConnected customers exposed to same external shocksCompetitor launches a feature that attracts an entire vertical

Part 2: The Identification Problem

The fundamental challenge: in observational data, you cannot directly distinguish contagion from homophily. Manski (1993) formalized this as the "reflection problem".

MANSKI'S REFLECTION PROBLEM
Yi = α + β · Ȳpeers + γ · X̄peers + δ · Xi + εi
β = endogenous peer effect (contagion) | γ = contextual peer effect (homophily) | Cannot separate without instruments
Why Naive Correlations Fail

You observe that customers connected to churners churn at 2x baseline. This could mean: (a) churn is contagious (β > 0), (b) connected customers share risk factors (γ > 0), or (c) both. A naive regression of "did i churn" on "did i's contacts churn" captures all mechanisms and overstates the true peer effect. To isolate the causal effect, you need an instrumental variable.

Part 3: Instrumental Variables for Peer Effects

An instrumental variable (IV) must be: (1) correlated with the suspected cause (peer's churn), (2) affect the outcome only through that cause, and (3) uncorrelated with unobserved confounders.

Instrument CandidateLogicStrength
Peer's contract expiry dateExogenous timing shock triggering peer's churnStrong if dates set independently
Random service outage in peer's regionExogenous shock to peer's satisfactionStrong natural experiment
Peer's acquisition channelAffects peer quality; shouldn't directly affect your behaviorModerate
The Two-Stage Approach (2SLS)

Stage 1: Regress peer's churn on instrument(s) + covariates. Get predicted (exogenous) peer churn.

Stage 2: Regress your churn on predicted peer churn + your covariates. The coefficient is the causal peer effect, purged of homophily.

Network-Adjusted CLV

NETWORK CLV
CLVnetwork = CLVdirect + ∑j ∈ neighbors β · CLVj
β = causal peer effect from 2SLS | For reference customers, network CLV can be 5-20x direct CLV

Part 4: Implementation in Python

Python · IV Peer Effects
import pandas as pd
import numpy as np
import networkx as nx
from linearmodels.iv import IV2SLS

# === Build Customer Network ===
edges = pd.read_csv('customer_connections.csv')
G = nx.from_pandas_edgelist(edges, 'cust_a', 'cust_b')
customers = pd.read_csv('customers.csv', index_col='id')

# === Peer Variables ===
for cid in customers.index:
    nbrs = list(G.neighbors(cid)) if cid in G else []
    if nbrs:
        customers.loc[cid, 'peer_churn'] = customers.loc[
            customers.index.isin(nbrs), 'churned'].mean()
        customers.loc[cid, 'peer_expiry'] = customers.loc[
            customers.index.isin(nbrs), 'contract_expiring'].mean()
    else:
        customers.loc[cid, 'peer_churn'] = 0
        customers.loc[cid, 'peer_expiry'] = 0

# === 2SLS IV Regression ===
model = IV2SLS(
    dependent=customers['churned'],
    exog=customers[['monthly_charges', 'tenure', 'nps']],
    endog=customers[['peer_churn']],
    instruments=customers[['peer_expiry']]
)
res = model.fit()
print(res.summary)
peer_effect = res.params['peer_churn']

# === Network-Adjusted CLV ===
customers['n_connections'] = [G.degree(c) if c in G else 0
                              for c in customers.index]
customers['network_clv'] = (
    customers['direct_clv'] +
    customers['n_connections'] * peer_effect *
    customers['direct_clv'].mean()
)
print(customers.nlargest(10, 'network_clv')[
    ['direct_clv', 'network_clv', 'n_connections']])
