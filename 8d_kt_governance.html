Governance & Reference

Quality Gates, Anti-Patterns
& Case Studies

The reviewer's guide to spotting analytical failures before they become closed 8Ds that reopen. Includes rejection criteria, the nine most common anti-patterns, escalation norms, and three real-world case studies.

Reviewer Rejection Criteria by Stage

Reviewers must reject stage exit if ANY of the following conditions apply. Document specific, actionable rationale — "needs more work" is not a valid rejection reason.

StageRejection TriggerRequired Fix Before Resubmission
D2 Problem statement is vague or unmeasurable • No baseline or quantification • Evidence is missing or anecdotal Rewrite with specific metrics, attach quantitative evidence, define IS/IS NOT boundaries
D4 Root cause does not explain all IS observations • Less than 3 independent evidence sources • No Tier 1 evidence • Less than 3 hypotheses scored in MPC • Escape cause missing Gather additional evidence, re-test hypothesis with structured protocol, complete MPC scoring
D5 Solution does not explicitly address root cause • No testability plan • Unacceptable new risks introduced • No pre-implementation success criteria Redesign solution with explicit root cause mapping and risk mitigation plan
D7 Prevention actions do not address escape cause • No ownership or timeline assigned • Vague action items ("improve monitoring") Specify concrete, measurable prevention actions with named owners and success criteria

Reviewer 2-Minute Scan Checklist

Run this before every D4 review gate. If ANY item fails → REJECT with specific feedback.

Problem statement is quantified and bounded (not vague descriptions)

IS/IS NOT has all 4 dimensions completed with DISTINCTION insights identified

Change Analysis timeline present with relevance assessment for each change

Minimum 3 hypotheses considered, MPC scoring documented with rationale

Root cause has ≥3 independent evidence sources (check tiers — Tier 1 required)

Both OCCURRENCE and ESCAPE root causes identified and validated

Solution directly addresses validated root cause (not just symptoms)

Prevention actions are specific, measurable, and have named owners

No anti-patterns from the list below detected in the analysis

Common RCA Failure Patterns & Remedies

These nine anti-patterns account for the majority of 8D rejections and repeat incidents. Reviewers should actively look for these; analysts should self-check before submitting.

Solution-First Bias
Symptoms

Team jumps from D2 to D5 • Solution proposed before root cause is validated • "We know what to do"

Remedy

Force D4 gate review. Demand: "What evidence proves this solution addresses the root cause, not the symptom?"

Correlation = Causation
Symptoms

"X happened, then Y happened, so X caused Y" • No mechanism explained • Timeline coincidence used as proof

Remedy

Require: (1) Mechanism explanation, (2) Lab reproduction or log trace, (3) Elimination test

Single-Source Evidence
Symptoms

Only 1 data source cited • "The logs show..." • Three log queries from the same system presented as 3 sources

Remedy

Require minimum 2 independent evidence sources (at least one Tier 1). Same system = same source.

Tribal Knowledge Dependence
Symptoms

"We think it is because..." • "Based on past experience..." • No documented evidence, expert opinion used as validation

Remedy

Document the source or discard. Expert opinion is Tier 3 — it can guide hypotheses, not validate root causes.

Escape Cause Ignored
Symptoms

Only OCCURRENCE root cause analyzed • No answer to "why did this reach customers?" • Detection gap not addressed

Remedy

Mandate BOTH root causes. Rejection trigger if escape cause is missing at D4 gate.

Vague Solutions
Symptoms

"Improve monitoring" • "Better testing" • "Enhance the process" • No WHO does WHAT by WHEN

Remedy

Require: WHO does WHAT by WHEN with WHICH measurable success criterion. Unmeasurable actions are rejected.

Partial IS/IS NOT
Symptoms

Only WHAT dimension filled • WHERE/WHEN/EXTENT missing • No DISTINCTION analysis performed

Remedy

Require all 4 dimensions completed. Reject if incomplete. Partial IS/IS NOT produces incomplete hypotheses.

Hypothesis Fatigue
Symptoms

First hypothesis declared root cause without testing alternatives • No MPC scoring • No alternative hypotheses considered

Remedy

Require minimum 3 candidate hypotheses with MPC scoring before accepting any root cause.

Premature Closure
Symptoms

Metrics improve slightly then 8D is closed • Containment mistaken for permanent solution • D7 prevention actions incomplete

Remedy

Track metrics 7+ days post-solution. Verify containment removed. Audit D7 prevention actions for completion.

Case Studies Across Industries

Telecom • Network Performance

5G Call Drop Rate Spike Post-Upgrade

Problem: Call drop rate increased from 1.2% to 8.5% in Region South after 5G upgrade.

KT Findings: IS: 5G calls, Region South, metro areas, peak hours. IS NOT: 4G calls, other regions, rural, off-peak. Change: New 5G core software v2.3 deployed Feb 1. Distinction: Region South has highest concurrent session density. MPC: Software bug in session management under high concurrency (scored 23/25).

Root Cause: Race condition in session cleanup logic triggered at >10K concurrent sessions — only occurs in high-density areas.

Outcome: Software hotfix with mutex lock • Call drop rate returned to 1.1% in 72 hours
Retail • Checkout Analytics

Mobile Checkout Abandonment Surge

Problem: Checkout abandonment jumped from 22% to 40% for mobile users after payment gateway update.

KT Findings: IS: Mobile Safari users, payment step, devices <4GB RAM. IS NOT: Desktop browsers, Android, iOS 15 and below. Change: New payment gateway SDK deployed Feb 1 with memory-intensive 3D Secure v2. Distinction: 3D Secure v2 requires 800MB RAM at peak — older devices fail silently.

Root Cause: Memory leak in 3D Secure v2 SDK causes payment iframe crash on constrained devices. Escape: No device performance testing in QA matrix.

Outcome: Lazy-loaded 3D Secure module • Abandonment returned to 23% in 48 hours
Banking • Transaction Processing

Auto-Debit Failure Rate 5x Spike

Problem: Auto-debit failure rate increased from 0.8% to 4.2% following payment processor migration.

KT Findings: IS: Recurring payments, credit card customers, amounts >$500. IS NOT: One-time payments, debit cards, amounts <$500. Change: New payment processor uses stricter velocity checks and different authentication flow for large recurring amounts.

Root Cause: New processor's fraud velocity check blocks recurring amounts >$500 without pre-authorization whitelist. Migration team did not migrate whitelist config.

Outcome: Whitelist migrated • Failure rate returned to 0.9% next day • $180K in late fees reversed
